Data Collection Strategy

We researched every platform's API. Here's what's actually available and our proposed ingestion approach.

Platform API Status Auth What We Get Our Plan
Ghost Full API JWT (Admin Key) Views, clicks, members, posts, newsletters API direct
LinkedIn Gated OAuth 2.0 + approval Post analytics, impressions (if approved) Playwright daily cron
Substack No API N/A No official endpoints Playwright daily cron
Medium Dead N/A API archived Mar 2023 Playwright daily cron or drop

Playwright MCP — Universal Fallback

For platforms without APIs, we run a Playwright MCP server on a daily cron. It logs into each analytics dashboard, scrapes the numbers, writes structured data to Airtable. One request per platform per day.

Why this works at our scale

  • 1 req/platform/day — barely detectable
  • Near-zero cost (headless browser on cron)
  • Universal — any platform with a web dashboard
  • No API approval or rate limits
  • Real browser fingerprint

Risk: botting detection

  • Platforms can flag automated logins
  • LinkedIn most aggressive at detection
  • Account suspension possible if caught
  • 1x/day is low risk but non-zero
  • Fallback: manual XLS/CSV export

Recommendation: Ghost via API. LinkedIn, Substack, Medium via Playwright MCP daily cron. If LinkedIn API approval comes through, migrate off Playwright. API is always better long-term — Playwright is our bridge to get data flowing from day one.
