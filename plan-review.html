Internal API approach (Plan B)

  • Lightweight — plain HTTP GET, no browser needed
  • Fast execution, low resource usage
  • Inspect DevTools Network tab to find endpoints
  • Replay GET calls with session cookies
  • Works like an unofficial API

Risks to manage

  • Origin URL / referer checking on some platforms
  • Session cookies expire — need refresh strategy
  • Endpoints may change without notice
  • CORS restrictions if calling from browser
  • Fallback: escalate to Crawlee or Playwright

Confirmed approach: Ghost via official API. LinkedIn, Substack, Medium — reverse-engineer internal API endpoints first (inspect browser network requests, replay with session cookies). If that fails, use Crawlee (like Scrapy but for Node.js). Playwright browser automation is the last resort. LinkedIn official API application runs in parallel.

Risk Register

CRIT

Medium API is dead

API archived March 2023. No programmatic path.

Plan: Reverse-engineer internal API endpoints. Fallback: Crawlee scrape, then drop from scope.

HIGH

Internal API session management

Session cookies expire. Origin/referer checking may block replayed requests. Endpoints can change without notice.

Mitigated by: cookie refresh strategy, request header spoofing. Fallback: escalate to Crawlee or Playwright.

HIGH

LinkedIn API approval timeline

Analytics API requires OAuth + case-by-case approval. Could take weeks.

Plan: Internal API scraping now, official API application in parallel, migrate when approved.

MED

Substack has no official API

Internal endpoints may be discoverable via DevTools network inspection. Could break anytime.

Plan: Internal API replay first, Crawlee second. Fallback: RSS + manual.
